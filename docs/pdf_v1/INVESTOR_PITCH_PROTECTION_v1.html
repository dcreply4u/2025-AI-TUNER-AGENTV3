## Red Flags Checklist

### Investor Red Flags üö©

- [ ] Refuses to sign NDA
- [ ] Wants code before serious discussion
- [ ] Asks for full repository immediately
- [ ] Vague about investment amount/timeline
- [ ] No track record or references
- [ ] Competitor or potential competitor
- [ ] Wants to "test" code themselves
- [ ] Pressure to share quickly
- [ ] Unusual requests
- [ ] Won't provide references

### NDA Red Flags üö©

- [ ] Vague confidentiality definition
- [ ] Short duration (<1 year)
- [ ] No return/destruction clause
- [ ] Allows reverse engineering
- [ ] One-sided (only protects them)
- [ ] No remedies clause
- [ ] Vague purpose
- [ ] No code-specific protections

## Negotiation Tips

### If Investor Resists NDA

**Response:**
"We understand your position. However, as a technology company with proprietary algorithms and trade secrets, we require an NDA before sharing technical details. This is standard practice and protects both parties. We're happy to discuss business aspects without an NDA, but technical discussions require protection."

### If NDA Terms Are Unfavorable

**Negotiate:**
- Duration (2-5 years is standard)
- Scope (limit to what's necessary)
- Exclusions (public information)
- Remedies (injunctive relief)
- Code-specific protections

**Don't Accept:**
- Perpetual confidentiality (unless trade secrets)
- One-sided terms (unless you have leverage)
- Vague definitions
- No remedies
